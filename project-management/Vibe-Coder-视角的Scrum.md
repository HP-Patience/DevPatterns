# Vibe Coder 视角的 Scrum：你需要吗？怎么用？

> 场景假设：你一个人（或 2-3 人小团队），用 AI 工具（Lovable / Cursor / Claude）快速搭建落地页、MVP、工具站。
> 看完上一篇 Scrum 标准框架后，你可能在想："这些会议、角色、工件，我一个开发搞定一切，有必要吗？"

直说答案：**不需要照搬，但 Scrum 的内核对你非常有用。**

---

## 一、先看本质：Scrum 到底在解决什么？

Scrum 诞生的源头是瀑布式的三个致命问题：

| 瀑布式的问题 | 本质原因 |
|-------------|---------|
| 做完才发现不对 | 需求前期定死，没有中间反馈 |
| 改动代价巨大 | 变更要重新走全流程 |
| 团队扯皮严重 | 角色模糊、责任不清 |

Scrum 的解法：**快反馈 + 小批量 + 固定节奏**。

你不是在"大公司跨部门协作写 ERP 系统"，你是一个人（或两三个）用 AI 快速出活的做页面、做 MVP，瀑布的问题是**不存在**的——因为你本来就在快速迭代，没人卡你流程。

所以 **Scrum 的角色、会议、工件体系对你来说是过度工程**。

但它的**核心节奏感**，对 Vibe Coder 帮助巨大。

---

## 二、Vibe Coder 的最大问题不是流程，而是"飘"

一个人用 AI 做项目最常见的翻车链：

```
灵感来了 -> 让 AI 干一票 -> 效果不错 -> 再加功能 ->
AI 代码开始乱 -> 改不动了 -> 重开项目 -> 循环
```

这不叫敏捷，这叫**无头苍蝇模式**。问题不在 AI，在于：

1. **没有固定节奏**——今天干 10 小时，明天不想动
2. **没有范围控制**——做着做着加需求，永远做不完
3. **没有验收标准**——自己觉得"差不多了"就停，后面发现到处是坑
4. **AI 代码债务**——没有回头清理，越堆越重

---

## 三、从 Scrum 抽三个最适合你的东西

### 1. Sprint —— 设定时间盒

不要"做完为止"，改为**两周固定周期**（或一周，看你节奏）。

- 时间一到，停新功能，只修 Bug + 写笔记
- 不要完美主义，能用就放
- 这是对抗"做不完"的最好武器

Vibe 用法：Sprint 规划时，把任务写成让 AI 能直接执行的粒度。比如不写"实现用户系统"，而写"用 Supabase 建 users 表 + 注册页面 + 邮箱验证"——AI 一次性理解边界，产出更干净。

### 2. Sprint Goal —— 每个周期只说清楚一件事

```
❌ 这周我要做一个 landing page + 用户系统 + 后台面板 + SEO 优化

✅ 这周先让落地页上线，能收集用户邮箱就行
```

一个 Goal 做完再做下一个。不是一个 Sprint 内只做一件事，而是所有任务服务于同一个关键目标。

### 3. Retro —— 回头看看 AI 挖了什么坑

每两周结束前，花 20 分钟问自己：

- 这轮 AI 生成的代码有什么重复模式让我改了很久？
- 有没有 prompt 模版可以复用提升下次效率？
- 哪些功能被 AI 搞得太复杂，其实可以删掉？
- 下一次有什么工作流可以优化？

→ 把结论写下来。下次启动 AI 时先把笔记丢给它，上下文质量会翻倍。

---

## 四、给你一套轻量级 Vibe Sprint 模板

### 周期：7 天（一周一迭代）

| 时间 | 事件 | 具体做啥 |
|------|------|---------|
| 周一 30min | 规划 | 定本周 Goal + 拆 3-5 个清晰任务 |
| 每日 5min | 自站会 | 回顾昨天 AI 输出 + 今天要 AI 做什么 |
| 周五 20min | 回顾 | 清理代码坑 + 总结有效 Prompt + 写笔记 |
| 周末 | 休息 | 不动代码 |

### 核心原则

1. **一个 Goal**——这周只服务一个明确目标
2. **小任务**——每个任务 AI 在 1-2 次对话内能干完
3. **留清理时间**——每周五留时间还技术债
4. **记 Prompt**——好的 prompt 是复利资产
5. **不插队**——这周的目标中途不变，新想法记下来下周再干

---

## 五、总结：Vibe Coder 正确吸收 Scrum

| Scrum 元素 | 要不要 | 怎么简化 |
|-----------|--------|---------|
| PO | 不需要 | 你就是 PO，但要写清楚"做什么"再开干 |
| SM | 不需要 | 一个人不需要流程教练 |
| 开发团队 | 你就是 | AI 是你的"结对编程搭档" |
| Sprint Backlog | 推荐 | 简单列个清单即可，不用工具 |
| Sprint Planning | 推荐 | 每周一 30 分钟定方向 |
| Daily Scrum | 看情况 | 节奏乱了就做，稳定了可免 |
| Sprint Review | 看情况 | 有外部反馈就做，没有跳过 |
| Retro | **强烈推荐** | 这是你迭代自己工作流的关键步骤 |
| Sprint 节奏 | **强烈推荐** | 对抗无头苍蝇模式的王牌武器 |

---

**最后一句**：Scrum 本质上是一套"防飘系统"。Vibe Coding 让你飞得快，Scrum 的核心节奏让你不飞偏。

不是用它的流程，是用它的节奏。
