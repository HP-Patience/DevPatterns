# 信息采集 SOP — 视频篇

**核心理念**：注意力是稀缺资源。不追求"看完"，追求"关键信息快速提取 + 可回溯"。

---

## 一、采集流程

```
刷到视频 → 丢链接到 Inbox
  ↓（集中处理时段）
① 抖虫提词 → https://www.dousnap.com/
  ↓
② 抖虫总结 → https://www.dousnap.com/video-summary
  ↓
③ 写入信息源笔记（URL / 文案 / 总结）
  ↓
④ AI 读取信息源 → 输出汇报（含 [[wikilinks]]）
  ↓
⑤ 信息源移入 Archive
```

---

## 二、各步骤操作规范

### 0. Inbox — 刷到视频先丢链接

- 打开 `Inbox.md`
- 刷视频时看到感兴趣的，**只粘贴 URL**，一行一个
- 不做任何处理，不打断当前状态
- 目标：把"发现"和"处理"解耦

### ① 集中处理：提词（视频 → 文案）

- 打开 Inbox.md，从上到下逐条处理
- 去 https://www.dousnap.com/ 粘贴视频链接，获取完整文案
- 快速扫读，判断是否值得进入下一步（价值过滤，节省注意力）

### ② 视频总结

- 打开 https://www.dousnap.com/video-summary
- 粘贴同一链接，获取**AI 视频总结**
- 这一步产出**结构化摘要**，让你不用看原视频也能掌握核心

### ③ 写入信息源笔记

在 `信息源/` 目录下新建笔记，格式如下：

```markdown
---
source: 抖音 / B站 / YouTube
date: {{采集日期}}
status: waiting / reported
---

url：{{视频链接}}

# 文案：

{{抖虫提取的完整文案}}

# 总结：

{{抖虫生成的视频总结}}
```

> **命名规则**：`{{平台}}-{{核心主题}}.md`
> 例：`抖音-渐进式披露.md`、`B站-RAG系统架构.md`

### ④ AI 汇报

AI 读取 `信息源/` 目录下的笔记后，按以下格式产出汇报：

- 每条信息标注来源（[[视频笔记名]] 双链）
- 跨笔记提炼共性模式
- 给出可执行的行动建议

汇报完成后，将该信息源笔记的 frontmatter 中 `status` 改为 `reported`。

### ⑤ 归档 Archive — 已汇报的信息源移入 Archive

- 已处理完、已汇报过的信息源笔记从 `信息源/` 移入 `Archive/`
- 清理 Inbox 中对应该视频的 URL 条目（或标记为 `✅`）
- 信息汇报保留在 `信息汇报/`，不移入 Archive

> 原则：信息源是"正在处理"，Archive 是"已完结"。保持信息源目录干净，只看待办。

## 三、目录结构

```
Obsidian Vault/
├── 07-信息输入/            ← 信息采集总入口
│   ├── Inbox.md            ← 刷到视频先丢 URL，一行一个
│   ├── 信息源/             ← 正在处理的笔记（status: waiting）
│   ├── 信息汇报/           ← AI 产出的分析汇报
│   ├── Archive/            ← 已汇报完结的信息源（status: reported）
│   └── 信息采集SOP-视频篇.md
```

---

## 四、价值过滤原则

不是每个视频都值得走完4步。在步骤①后快速判断：

| 判断 | 动作 |
|------|------|
| 文案看完觉得水/无新信息 | 丢弃，不进入下一步 |
| 有1-2个亮点但整体一般 | 只写URL+总结，不存全文文案 |
| 高质量信息 | 完整走完流程（①→⑤），进AI汇报 |

---

## 五、AI 汇报 Prompt 模板

```markdown
请阅读 [[信息源/]] 目录下的所有笔记，为我做一份信息汇报：

1. 每条信息的核心观点（附 [[wikilink]] 来源）
2. 跨信息之间的共性模式或矛盾点
3. 对我有价值的行动建议
4. 哪些信息值得深入跟进
```
